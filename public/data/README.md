# Данные для сайта

## Расписание намазов (`prayer-times.json`)

Сайт показывает время намазов **на сегодня и завтра** из этого файла. Нужный день подставляется автоматически по текущей дате — **обновлять что-то каждый день не нужно**. Достаточно один раз в месяц загрузить расписание на весь месяц.

### Как обновить расписание на месяц

**Способ 1 — из CSV (таблица из Excel или с сайта расписания)**  
Сохраните таблицу в CSV с колонками: **день** (1–31) или **дата** (ГГГГ-ММ-ДД), **фаджр**, **зухр**, **аср**, **магриб**, **иша** (время ЧЧ:ММ). Из корня проекта:

```bash
# колонка «день» в CSV — укажите месяц и год
node scripts/csv-to-prayer-times.js --month=2 --year=2026 ваш-файл.csv

# в CSV уже есть колонка «дата»
node scripts/csv-to-prayer-times.js ваш-файл.csv
```

Скрипт запишет результат в `public/data/prayer-times.json`. Опция `--merge` добавляет новые даты к существующему расписанию.

**Способ 2 — вручную**  
1. Откройте файл `prayer-times.json`.  
2. В объекте **`schedule`** добавьте или измените записи по датам.
3. **Ключ** — дата в формате **ГГГГ-ММ-ДД** (например, `2025-02-17`).
4. **Значения** — время каждого намаза в формате **ЧЧ:ММ** (24 часа):

```json
"schedule": {
  "2025-02-17": {
    "fajr": "05:30",
    "dhuhr": "13:15",
    "asr": "16:45",
    "maghrib": "19:20",
    "isha": "21:00"
  },
  "2025-02-18": { ... }
}
```

- **fajr** — Фаджр  
- **dhuhr** — Зухр  
- **asr** — Аср  
- **maghrib** — Магриб  
- **isha** — Иша  

5. Сохраните файл и обновите страницу сайта. Если файл лежит в `public/data/`, можно просто заменить его на новый (например, загрузить готовый JSON за месяц).

### Если даты нет в файле

Для даты без записи в `schedule` сайт покажет время по умолчанию (примерное). Чтобы всегда было актуально, добавляйте в файл все нужные дни.
